Java并发编程——深入了解FutureTask

发表于 2018-06-19 | 分类于 并发编程 | 阅读次数：

FutureTask类实现了RunnableFuture<V>接口，而RunnableFuture<V>接口继承了Runnable接口和Future<V>接口。Runnable接口可以被另一个线程执行，而Future<V>接口代表异步计算的结果，通过Future接口提供的方法可以查看异步计算是否执行完成，或者等待执行结果并获取执行结果，同时还可以取消执行。所以FutureTask既能当做一个Runnable直接被Thread执行，也能作为Future用来得到Callable的计算结果。

Runnable接口

Runnable接口的定义：

实现了Runnable接口的对象可用来创建线程，调用线程的start方法会在该线程中调用run方法。

Future接口

Future接口的定义：

它定义了5个方法：

  • cancel方法：
    用来取消任务的执行。如果任务已经结束，或已经被取消，或者由于某种原因不能取消，则取消任务失败，返回false。如果取消任务成功，并且该任务在调用cancel方法时未开始执行，则该任务永远不会执行。如果任务已经开始，则参数mayInterruptIfRunning表示是否能取消正在执行却没有执行完毕的任务。
  • isCancelled
    如果任务在正常完成之前被调用cancel方法取消，则返回true
  • isDone
    表示该任务是否结束，包括：正常完成，抛出异常或者被取消
  • get()
    获取任务执行的结果，如果任务还未完成，则该方法阻塞。
  • get(long timeout, TimeUnit unit)
    用来获取执行结果，如果在指定时间内，还没获取到结果，就直接返回null

‘private volatile int state;’变量用来保存任务的执行状态，它是volatile的，因此各线程可见（即一个线程修改了state，则其他线程可以看到state的变化），它有7中状态：

可能的状态转换：

NEW（初始状态）：表示是个新的任务或者还没被执行完的任务。
COMPLETING（中间状态）：任务已经执行完成或者执行任务的时候发生异常，但是任务执行结果或者异常原因还没有保存到outcome字段(outcome字段用来保存任务执行结果，如果发生异常，则用来保存异常原因)的时候，状态会从NEW变更到COMPLETING。
NORMAL（终态）：任务已经执行完成并且任务执行结果已经保存到outcome字段，状态会从COMPLETING转换到NORMAL。
EXCEPTIONAL（终态）：任务执行发生异常并且异常原因已经保存到outcome字段中后，状态会从COMPLETING转换到EXCEPTIONAL。
CANCELLED（终态）：任务还没开始执行或者已经开始执行但是还没有执行完成的时候，用户调用了cancel(false)方法取消任务且不中断任务执行线程，这个时候状态会从NEW转化为CANCELLED状态。
INTERRUPTING（中间状态）：任务还没开始执行或者已经执行但是还没有执行完成的时候，用户调用了cancel(true)方法取消任务并且要中断任务执行线程但是还没有中断任务执行线程之前，状态会从NEW转化为INTERRUPTING。
INTERRUPTED（终态）：调用interrupt()中断任务执行线程之后状态会从INTERRUPTING转换到INTERRUPTED。

get()

任务发起线程可以调用get()方法来获取任务执行结果，实现如下：

如果任务还未执行完成，则会调用awaitDone方法，在该方法中阻塞。否则会调用report方法返回执行结果。
看一下awaitDone方法的实现，该方法等待任务完成，或在线程被中断或超时时丢弃任务，在get()方法中的传参为awaitDone(false, 0L)：

该方法中有一个死循环用于方法的阻塞，即for (;;)。
FutureTask维护了一个变量private volatile WaitNode waiters;，waiters用来保存执行该任务的等待线程，用一个链表实现（Treiber stack）。因为同一个FutureTask对象可能会在多个线程中调用get()方法，如果任务执行时间很长，则可能会存在多个等待任务执行的线程。

  1. 判断调用get()的线程是否被其他线程中断，如果是的话则在等待队列中删除对应节点然后抛出InterruptedException异常。
  2. 获取任务当前状态，如果当前任务状态大于COMPLETING则表示任务执行完成，则把thread字段置null并返回结果。
  3. 如果任务处于COMPLETING状态，则表示任务已经处理完成(正常执行完成或者执行出现异常)，但是执行结果或者异常原因还没有保存到outcome字段中。这个时候调用线程让出执行权让其他线程优先执行。
  4. 如果等待节点为空，则构造一个等待节点WaitNode，注意WaitNode的构造函数，将thread置为当前线程。
  5. 如果第四步中新建的节点还没如队列，则CAS的把该节点加入waiters队列的首节点，相当于入栈。
  6. 阻塞等待，直到任务执行完成被唤醒。

总结

FutureTask内部维护一个任务状态，任何操作都是围绕着这个状态进行，并随时更新任务状态。任务发起者调用get()获取执行结果的时候，如果任务还没有执行完毕，则会把自己放入阻塞队列中然后进行阻塞等待。当任务执行完成之后，任务执行线程会依次唤醒阻塞等待的线程。调用cancel()取消任务的时候也只是简单的修改任务状态，如果需要中断任务执行线程的话则调用Thread.interrupt()中断任务执行线程。
